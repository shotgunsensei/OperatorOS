2. Domain migration documentation

Create:

Include step-by-step instructions for Tech Deck migration while using Replit as current host.

Required sections:
- Purpose
- Current model
- Target model
- Manual Replit steps
- Manual DNS steps
- Verification steps
- Legacy redirect steps
- Rollback notes
- Do-not-redirect warning

The doc must explain this exact flow:

1. Open Replit project.
2. Go to Publishing / Deployments / Domains, using the naming that best matches current Replit UI.
3. Add custom domain:
   techdeck.operatoros.net
4. Copy the DNS records Replit provides.
5. Add the generated A/TXT/CNAME records at the DNS provider for operatoros.net.
6. Wait for Replit to show verified.
7. Test DNS resolution.
8. Test HTTPS:
   https://techdeck.operatoros.net
9. Confirm the app loads successfully over HTTPS.
10. Only after successful HTTPS testing, redirect:
    techdeck.app → https://techdeck.operatoros.net
11. Keep techdeck.app as a legacy alias or redirect.

Explicit warning text required:

Do not redirect techdeck.app until https://techdeck.operatoros.net resolves correctly and serves HTTPS successfully. Redirecting early can break the current public entry point.
