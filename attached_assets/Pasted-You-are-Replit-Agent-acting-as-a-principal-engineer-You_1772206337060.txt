4) Runner Gateway (apps/runner-gateway)
Implement:
- POST /v1/runner/provision { workspaceId }:
  - Looks up workspace in DB (via api call or shared DB access; choose one and keep consistent)
  - Creates PVC `ws-<id>` in namespace veridian-runners
  - Creates Pod `ws-<id>` with:
    initContainer: alpine/git cloning git_url at git_ref into /workspace if empty
    main container: profile image, command `sleep infinity` (so it stays alive)
    volume mount: /workspace
    resources limits: cpu/mem reasonable
- POST /v1/runner/stop { workspaceId } -> delete pod (keep PVC)
- GET  /v1/runner/status/:workspaceId -> pod phase + ready status + last start time
- POST /v1/runner/exec { workspaceId, cmd, timeoutSec? }:
  - Validate cmd against denylist unless ALLOW_UNSAFE_COMMANDS=true
  - Execute inside pod at /workspace (use k8s exec stream or kubectl fallback)
  - Capture stdout/stderr (truncate to 1MB each)
  - Emit WS events to subscribers on channel workspaceId
  - Return JSON { exitCode, stdout, stderr, durationMs, truncated }
- WS /v1/runner/stream/:workspaceId:
  - Clients subscribe; keep a map workspaceId -> set of sockets; broadcast stdout/stderr/status events.

Also implement a docker fallback runner mode (only if k3d unavailable):
- Create per-workspace docker container named ws-<id> from profile image, mount a local volume path to /workspace, clone git if empty, exec via docker exec.
- Abstract runner interface: RunnerProvider { provision, stop, status, exec } with K8sRunnerProvider and DockerRunnerProvider; select via env RUNNER_MODE=k8s|docker.

5) Control Plane API (apps/api)
Implement Fastify API with routes:
- POST /v1/workspaces { gitUrl, gitRef, profileId } -> create workspace record status=CREATED
- POST /v1/workspaces/:id/start -> call runner-gateway /v1/runner/provision ; update status=RUNNING or ERROR
- GET  /v1/workspaces/:id -> workspace + runner status
- POST /v1/workspaces/:id/exec { cmd, timeoutSec? } -> proxy to runner-gateway exec
- POST /v1/workspaces/:id/apply-patch { diff }:
  - Validate unified diff:
    - max 20KB
    - deny paths: .env*, *.pem, *.key, node_modules/**, dist/**, build/**, .git/**, package-lock.json? (allow), pnpm-lock.yaml? (allow), yarn.lock? (allow)
  - Apply patch by calling runner exec: `bash -lc "cd /workspace && git apply --whitespace=nowarn -"` with diff piped in (implement safe temp file method if piping is hard)
  - Return git status porcelain + list of changed files
- POST /v1/workspaces/:id/git-status -> exec `git status --porcelain`
- POST /v1/workspaces/:id/create-branch { name } -> exec `git checkout -b <name>`
- POST /v1/workspaces/:id/commit { message } -> exec `git add -A && git commit -m "<message>"` (handle “nothing to commit”)
- POST /v1/workspaces/:id/verify -> runs verification plan from profile:
  - sequentially call runner exec for each check, store structured results in DB (workspace_ports used if preview)
  - return results
- Tasks:
  - POST /v1/tasks { workspaceId, title } -> create task required_checks from profile verify plan
  - POST /v1/tasks/:taskId/run -> execute deterministic task runner (packages/agent-runtime) that:
      - creates branch task/<id>
      - runs verify
      - records task_events + tool_traces
      - sets SUCCEEDED/FAILED
  - GET /v1/tasks/:taskId -> task record
  - GET /v1/tasks/:taskId/events -> ordered events
