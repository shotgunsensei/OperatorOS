You are working on the existing OperatorOS application.

Goal:
Transform the current frontend from a generic workspace/project/task SaaS dashboard into a polished multi-tenant software ecosystem control center for OperatorOS by Shotgun Ninjas.

OperatorOS is the central operating layer that houses and manages access to multiple software modules/products, including but not limited to:

- TradeFlowKit
- TorqueShed
- TechDeck
- PulseDesk
- FaultLineLab
- CallCommand AI
- SnapProofOS
- Ninjamation
- BrandForgeOS
- StudyForge AI
- Ninja Launch Kit
- Future modules added by super admin

The current UI still looks like a basic SaaS project manager with Workspaces, Projects, Tasks, Notes, and generic dashboard cards. Redesign it so OperatorOS feels like a premium SaaS ecosystem hub, product launcher, billing center, and admin command console.

Do not remove existing working backend functionality unless it is clearly obsolete or replaced safely. First inspect the existing codebase, routing, auth system, database schema, billing implementation, current module/add-on handling, and role model. Then implement this as a production-grade refactor.

Primary user roles:
1. Super Admin
   - Platform owner/operator.
   - Can manage all tenants, modules, billing visibility, users, feature flags, platform settings, announcements, product visibility, and audit logs.
2. Tenant Admin
   - Admin for a tenant/business account.
   - Can manage their tenant users, view purchased modules, purchase add-ons, manage billing, configure enabled products, and view usage.
3. Regular User
   - End user inside a tenant.
   - Can access assigned products/modules, see available tools, view account status where allowed, and request access or upgrades.

Critical requirement:
The UI must visibly and structurally communicate that OperatorOS is the parent ecosystem and the modules are the products. The dashboard should not lead with generic Workspaces/Projects/Tasks. Those can remain as secondary internal features if they are already part of the app, but they should not be the main identity of the platform.
